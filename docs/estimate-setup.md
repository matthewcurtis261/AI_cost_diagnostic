# Cost estimation (Estimate phase)

Turn discovery findings — and optionally telemetry events — into dollar estimates and savings comparisons.

---

## Prerequisites

1. **Findings JSON** from a scan (`ai-usage-findings.json`)
2. *(Optional)* **Telemetry JSONL** from the [Observe phase](telemetry-setup.md)

---

## Quick start — code-only mode

Assumes default token heuristics and **10,000 calls/month** per billable call site:

```bash
pnpm run estimate -- --findings examples/sample-findings.json
```

Custom volume:

```bash
pnpm run estimate -- \
  --findings test-runs/rival-search/ai-usage-findings.json \
  --default-model gpt-4o-mini \
  --calls-per-month 5000
```

Write JSON report:

```bash
pnpm run estimate -- \
  --findings examples/sample-findings.json \
  --output spend-estimate.json
```

Validate output:

```bash
pnpm run validate-estimate -- spend-estimate.json
```

---

## Telemetry mode (accurate volumes)

When telemetry is enabled in your app:

```bash
pnpm run estimate -- \
  --findings test-runs/rival-search/ai-usage-findings.json \
  --telemetry \
  --events ~/.diagnostic_agent/events.jsonl
```

Or pass an explicit events path:

```bash
pnpm run estimate -- \
  --findings ai-usage-findings.json \
  --events ./events.jsonl
```

Filter by time window:

```bash
pnpm run estimate -- \
  --findings ai-usage-findings.json \
  --telemetry \
  --since 2026-06-01T00:00:00Z \
  --until 2026-06-30T23:59:59Z
```

In telemetry mode, only findings with matching `finding_id` events are costed.

---

## Custom assumptions

For finer control without telemetry, use an assumptions file:

```bash
pnpm run estimate -- \
  --findings examples/sample-findings.json \
  --assumptions examples/assumptions.json
```

`examples/assumptions.json`:

```json
{
  "defaults": {
    "calls_per_month": 5000,
    "avg_input_tokens": 900,
    "avg_output_tokens": 350
  },
  "findings": {
    "f001": {
      "model": "gpt-4o-mini",
      "calls_per_month": 5000
    }
  }
}
```

## Savings comparison

Compare current models against cheaper alternatives (includes open models by default):

```bash
pnpm run estimate -- \
  --findings examples/sample-findings.json \
  --alternatives gpt-4o-mini,deepseek-chat,llama-3.1-8b-instruct
```

If `--alternatives` is omitted, the pricing table's `default_alternatives` are used.

The report includes `savings_opportunities` ranked by dollar savings.

---

## How billable call sites are selected

From findings, the estimator includes:

- `chat_completion`, `embedding`, `image`, `speech` call types
- Evidence of an actual API call (`.create(`, `chat.completions`, etc.)
- Deduplicated by file + call type (wrapper/delegate findings are merged)

Excluded by default:

- Dependency-only findings (`agent_framework` from `requirements.txt`)
- Client init / import-only hits
- Low-confidence hits

## Pricing table

Models and rates live in two files, **merged by default** when you run `estimate`:

| File | Contents |
|---|---|
| `estimate/pricing/models.json` | Proprietary models (OpenAI, Anthropic, Google) |
| `input-analysis/pricing/models-open.json` | Open/smaller models (DeepSeek, Llama, Qwen, Mistral, Phi) + self-hosted baseline |

| Field | Meaning |
|---|---|
| `input_per_million` | USD per 1M input tokens |
| `output_per_million` | USD per 1M output tokens |
| `deployment` | `api` or `self_hosted` (open models only) |
| `as_of` | Date pricing was last reviewed |

**Self-hosted baseline:** `$0.20/Mtok` input and output (see `self_hosted_compute` in `models-open.json`).

Override proprietary pricing:

```bash
pnpm run estimate -- --findings examples/sample-findings.json --pricing path/to/models.json
```

Override open pricing:

```bash
pnpm run estimate -- --findings examples/sample-findings.json --open-pricing path/to/models-open.json
```

Proprietary only (no DeepSeek/Llama savings):

```bash
pnpm run estimate -- --findings examples/sample-findings.json --no-open-pricing
```

The report's `estimate_metadata.pricing_sources` lists which files were merged.

## Output schema

`estimate/schema/estimate.schema.json`

Key sections:

| Section | Purpose |
|---|---|
| `line_items` | Per-finding cost breakdown |
| `totals` | Aggregated tokens and USD |
| `savings_opportunities` | Model swap recommendations |
| `coverage_notes` | Assumptions, missing pricing, skipped findings |

See `examples/sample-estimate.json` (generated by tests).

  --findings test-runs/rival-search/ai-usage-findings.json \

```bash

```

## CLI reference

```
diagnostic-agent estimate --findings <path> [options]

  --findings <path>         Required. Scan output JSON.
  --telemetry               Use telemetry mode
  --events <path>           Telemetry JSONL (default: ~/.diagnostic_agent/events.jsonl)
  --output <path>           Write spend-estimate.json
  --assumptions <path>      Custom volume/token assumptions
  --pricing <path>          Override proprietary pricing (models.json)
  --open-pricing <path>     Override open/self-hosted pricing (models-open.json)
  --no-open-pricing         Exclude open models from pricing and savings defaults
  --calls-per-month <n>     Default monthly calls (code-only)
  --default-model <id>      Resolve config_ref/dynamic models
  --alternatives <a,b,c>    Models for savings comparison
  --since / --until         Telemetry time filter (ISO-8601)
  --json                    Print full JSON to stdout
```
